# Skool Lesson: Build Your Second Brain AI OS

## Lesson Title

Build Your Second Brain: Turn Claude Into A Personal OS

## Short Description

Turn Claude Code into an operating system for your life and business. This starter folder gives Claude a home: context, decisions, goals, projects, references, and future skills.

## Repo

```text
https://github.com/mahmoud-hussein-dm/personal-os-starter
```

## What Members Will Build

By the end of this lesson, members will have:

- A local Personal OS folder
- A `CLAUDE.md` brain file
- An `AGENTS.md` file for coding agents
- Context files for their life and business
- A decision log
- A project folder system
- A place to build repeatable skills later

## Setup Steps

### 1. Install Claude Code

Install Claude Code through the CLI or VS Code extension.

### 2. Clone The Starter Repo

```bash
git clone https://github.com/mahmoud-hussein-dm/personal-os-starter my-personal-os
cd my-personal-os
```

### 3. Open The Folder In Claude Code

Open the whole folder, not a single file.

The folder is the brain. If you work outside it, the system will not compound.

### 4. Paste The Initialize Prompt

Open:

```text
Executive Assistant Initialize Prompt.txt
```

Paste this into Claude Code:

```text
Read Executive Assistant Initialize Prompt.txt and run Phase 1. Then interview me section by section for Phase 2.
```

### 5. Answer The Interview

Claude will ask about:

- Who you are
- What you do
- Your business/work
- Your team
- Your priorities
- Your goals
- Your communication style
- Recurring workflows you want help with

Do not rush this. The answers become the assistant's operating context.

### 6. Connect Tools Later

You do not need tools connected on day one.

Start with the folder. Then connect tools like:

- Gmail
- Notion
- Google Drive
- Shopify
- Meta Ads
- Telegram
- GitHub

### 7. Build Cadence

The system becomes valuable when you use it repeatedly:

- Ask it to update your priorities
- Log decisions
- Summarize sessions
- Track active projects
- Turn repeated workflows into skills

## The Rule That Makes It Work

Always work inside the folder.

That is what makes the assistant compound.

## What Breaks This

- Keeping context scattered across random chats
- Not updating priorities
- Not logging decisions
- Putting secrets into git
- Creating skills before you understand the workflow
- Treating the folder like a one-time setup instead of an operating system
